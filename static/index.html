Base URL 怎么填？
不同工具对 Base URL 的要求不同，以下三种写法都可以，按工具要求选：
• https://api.mofa.ai — 大多数工具（Cursor、ChatBox 等）
• https://api.mofa.ai/v1 — OpenAI SDK、部分工具要求带 /v1
• https://api.mofa.ai/v1/chat/completions — 少数需要完整端点的工具
如果不确定，先试第一种。

    FAQ

    这是什么？

    这是一个 API 中转服务。你可以用统一的地址和 Key 访问多家 AI 模型，无需分别注册各家服务。

    支持 streaming 吗？

    支持。在请求中加入 "stream": true 即可获得流式响应。

    支持哪些 API 格式？

    支持以下端点：

    • /v1/chat/completions — 文本对话（OpenAI 格式，支持 Kimi / Claude / Gemini）

    • /v1/messages — 文本对话（Anthropic 格式，支持 Claude）

    • /v1/files — 文件上传（用于图片/视频理解，支持 Kimi）

    • /v1/images/generations — 图片生成（支持 Imagen 4.0）

    • /v1/videos/generations — 视频生成（支持 Veo 2.0 / 3.0）

    支持图片上传吗？

    支持两种方式：1) 将图片 base64 编码后直接放在消息中（data:image/png;base64,...）；2) 先通过 /v1/files 上传文件，再用返回的 file ID（ms://FILE_ID）引用。单次请求最大 100MB。

    和直接调用官方 API 有什么区别？

    对你的代码来说没有区别——请求和响应格式完全一样，只是换了地址和 Key。

    API Key 安全吗？

    你的 Key 只用于本服务的身份验证，不会被发送到任何上游服务商。请妥善保管，不要分享给他人。
